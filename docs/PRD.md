# Product Requirement Document (PRD)
## Product Name
Aeroport Management System

## Product Type
Frontend Dashboard Application

## Scope
Frontend UI only. No backend or database integration.

## Core Requirement
- Create 6 dashboards.
- Each dashboard must contain exactly 8 menus.
- Each menu must open a different page layout.

## Dashboards

### 1. Admin Dashboard
- Overview
- Airport Management
- Aircraft Management
- Staff Management
- Flight Scheduling
- Reports
- Notifications
- System Settings

### 2. Pilot Dashboard
- Pilot Overview
- Assigned Flights
- Flight Logs
- Weather Reports
- Flight Schedule
- Messages
- Notifications
- Profile

### 3. Passenger Dashboard
- Passenger Overview
- My Tickets
- Booking History
- Flight Status
- Check-in
- Notifications
- Support
- Profile

### 4. Flight Operations Dashboard
- Operations Overview
- Flight Tracking
- Departure Management
- Arrival Management
- Gate Management
- Delay Monitoring
- Reports
- Alerts

### 5. Maintenance Dashboard
- Maintenance Overview
- Aircraft Status
- Inspection Logs
- Maintenance Schedule
- Repair Requests
- Spare Parts
- Reports
- Alerts

### 6. Ticketing & Booking Dashboard
- Booking Overview
- Ticket Booking
- Ticket Cancellation
- Payment Records
- Passenger List
- Flight Availability
- Reports
- Notifications

## UI Components
Every dashboard page must include:
- Sidebar Navigation
- Top Navigation Bar
- Dashboard Cards
- Tables
- Forms
- Charts
- Notifications Panel

## UI Design Style
- Modern aviation dashboard style
- Clean layout
- Dark sidebar
- Light content area
- Card based analytics
